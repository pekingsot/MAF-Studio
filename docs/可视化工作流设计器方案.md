# 可视化工作流设计器方案

## 🎯 核心理念

**从固定模式 → 自由设计**

用户通过拖拽Agent节点、连接节点、配置参数，来自由设计协作工作流。

---

## 🎨 节点类型

### 1. Agent节点

```
┌──────────────────┐
│  🤖 Agent节点     │
├──────────────────┤
│  名称：编写者     │
│  Agent: Agent1   │
│  角色：creator   │
└──────────────────┘
```

**配置项**：
- Agent选择
- 角色定义
- 输入提示词
- 超时设置
- 重试次数

---

### 2. 条件节点

```
┌──────────────────┐
│  🔀 条件判断      │
├──────────────────┤
│  条件：          │
│  output.contains │
│  ('[APPROVED]')  │
│                  │
│  ✅ True → 通过   │
│  ❌ False → 打回  │
└──────────────────┘

**配置项**：
- 条件表达式
- True分支
- False分支

---

### 3. 并发节点

```
┌──────────────────┐
│  ⚡ 并发执行      │
├──────────────────┤
│  策略：全部完成   │
│  超时：300秒      │

│  │A1│  │A2│  │A3││

- 并发策略（全部完成/任一完成）
- 最大并发数
- 超时时间
- 结果聚合方式

**推荐使用 React Flow**：

```bash
npm install react-flow-renderer
```

**示例代码**：

```typescript
import ReactFlow, { 
  Node, 
  Edge, 
  Controls, 
  Background,
  MiniMap 
} from 'react-flow-renderer';

const nodes: Node[] = [
  {
    id: '1',
    type: 'input',
    data: { label: '开始' },
    position: { x: 100, y: 100 },
  },
  {
    id: '2',
    type: 'agentNode',
    data: { 
      label: '编写者',
      agentId: 1,
      config: {}
    },
    position: { x: 300, y: 100 },
  },
  {
    id: '3',
    type: 'agentNode',
    data: { 
      label: '审阅者',
      agentId: 2,
      config: {}
    },
    position: { x: 500, y: 100 },
  },
  {
    id: '4',
    type: 'conditionNode',
    data: { 
      label: '条件判断',
      condition: 'output.contains("[APPROVED]")'
    },
    position: { x: 700, y: 100 },
  },
  {
    id: '5',
    type: 'output',
    data: { label: '结束' },
    position: { x: 900, y: 100 },
  },
];

const edges: Edge[] = [
  { id: 'e1-2', source: '1', target: '2' },
  { id: 'e2-3', source: '2', target: '3' },
  { id: 'e3-4', source: '3', target: '4' },
  { id: 'e4-5', source: '4', target: '5', label: '通过' },
  { id: 'e4-2', source: '4', target: '2', label: '打回' },
];

function WorkflowDesigner() {
  return (
    <div style={{ height: 600 }}>
      <ReactFlow
        nodes={nodes}
        edges={edges}
        fitView
      >
        <Controls />
        <Background />
        <MiniMap />
      </ReactFlow>
    </div>
  );
}
```

### 后端API设计

#### 1. 保存工作流

```http
POST /api/workflows
{
  "name": "文档审阅迭代流程",
  "nodes": [...],
  "edges": [...],
  "variables": {...}
}
```

#### 2. 执行工作流

```http
POST /api/workflows/{workflowId}/execute
{
  "input": "编写一份系统架构设计文档",
  "variables": {
    "maxIterations": 10
  }
}
```

#### 3. 获取执行状态

```http
GET /api/workflows/executions/{executionId}
```

#### 4. 获取执行历史

```http
GET /api/workflows/{workflowId}/executions
```

### 工作流引擎

**核心执行逻辑**：

```csharp
public class WorkflowEngine
{
    public async Task<WorkflowResult> ExecuteAsync(
        WorkflowDefinition workflow,
        string input,
        Dictionary<string, object> variables)
    {
        var context = new WorkflowContext
        {
            Input = input,
            Variables = variables,
            Outputs = new Dictionary<string, object>()
        };

        // 找到开始节点
        var startNode = workflow.Nodes.First(n => n.Type == "start");
        
        // 执行工作流
        await ExecuteNodeAsync(startNode, workflow, context);

        return new WorkflowResult
        {
            Success = true,
            Output = context.Output,
            Variables = context.Variables
        };
    }

    private async Task ExecuteNodeAsync(
        Node node, 
        WorkflowDefinition workflow, 
        WorkflowContext context)
    {
        switch (node.Type)
        {
            case "agent":
                await ExecuteAgentNodeAsync(node, context);
                break;
            case "condition":
                await ExecuteConditionNodeAsync(node, workflow, context);
                break;
            case "concurrent":
                await ExecuteConcurrentNodeAsync(node, workflow, context);
                break;
            case "loop":
                await ExecuteLoopNodeAsync(node, workflow, context);
                break;
        }

        // 执行下一个节点
        var nextNodes = GetNextNodes(node, workflow);
        foreach (var nextNode in nextNodes)
        {
            await ExecuteNodeAsync(nextNode, workflow, context);
        }
    }
}
```

## 🎯 总结

### 核心优势

1. **完全自定义** - 用户可以设计任何复杂的工作流
2. **可视化设计** - 拖拽式操作，直观易懂
3. **灵活扩展** - 可以随时添加新的节点类型
4. **实时预览** - 可以看到工作流的执行过程
5. **模板复用** - 可以保存和分享工作流模板

### 适用场景

✅ **强烈推荐**：
- 复杂的协作流程
- 需要条件分支的场景
- 需要循环迭代的场景
- 需要并发执行的场景
- 需要自定义流程的场景

### 技术选型

**前端**：
- React Flow（可视化流程设计）
- Ant Design（UI组件）
- TypeScript（类型安全）

**后端**：
- .NET 8（工作流引擎）
- PostgreSQL（工作流存储）
- SignalR（实时通信）
