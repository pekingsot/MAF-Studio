# 协作工作流使用指南

## 📋 概述

协作工作流功能允许你创建多Agent协作团队，并通过不同的工作流模式执行复杂任务。

## 🔄 完整使用流程

### 第一步：创建协作

1. **访问协作管理页面**
   - 在侧边栏点击"协作管理"
   - 或访问 http://localhost:3000/collaborations

2. **创建新协作**
   - 点击"新建协作"按钮
   - 填写协作信息：
     - **名称**：协作的名称（例如："代码审查团队"）
     - **描述**：协作的详细描述
     - **路径**：工作目录路径（可选）
     - **Git仓库信息**：如果需要Git协作（可选）

3. **保存协作**
   - 点击"确定"创建协作

### 第二步：添加Agent到协作

1. **在协作列表中找到刚创建的协作**
2. **点击"添加Agent"按钮**
3. **选择要添加的Agent**
   - 从下拉列表中选择Agent
   - 可以指定Agent的角色（例如："代码审查员"、"测试工程师"）
4. **重复此步骤添加多个Agent**
   - 建议至少添加2-3个Agent才能发挥协作优势

### 第三步：执行工作流

1. **访问协作工作流页面**
   - 在侧边栏点击"协作工作流"
   - 或访问 http://localhost:3000/collaboration-workflow

2. **选择协作**
   - 在协作列表中找到你创建的协作
   - 点击"执行工作流"按钮

3. **选择工作流类型**

   #### 🔹 顺序工作流（Sequential）
   - **适用场景**：流水线式任务处理
   - **执行方式**：Agent1 → Agent2 → Agent3 → 最终结果
   - **示例**：产品需求 → 架构设计 → 代码开发 → 测试

   #### 🔹 并发工作流（Concurrent）
   - **适用场景**：独立任务并行处理
   - **执行方式**：多个Agent同时执行，最后合并结果
   - **示例**：多个专家同时审查代码的不同方面

   #### 🔹 任务移交工作流（Handoffs）
   - **适用场景**：需要反复修改的场景
   - **执行方式**：Agent1 → Agent2 → Agent1 → Agent3（动态移交）
   - **示例**：文档编写 → 审核 → 修改 → 再审核

   #### 🔹 群聊工作流（Group Chat）
   - **适用场景**：需要多Agent讨论的场景
   - **执行方式**：多个Agent进行群聊协作
   - **示例**：团队讨论技术方案

4. **输入任务内容**
   - 在文本框中输入你的任务描述
   - 例如："请审查这段代码的安全性和性能"

5. **执行工作流**
   - 点击"执行工作流"按钮
   - 等待执行完成
   - 查看执行结果和过程消息

## 💡 实际使用示例

### 示例1：代码审查团队

**创建协作**：
```
名称：代码审查团队
描述：多角度审查代码质量
```

**添加Agent**：
- Agent1：代码逻辑审查员
- Agent2：代码风格审查员
- Agent3：安全审查员

**执行工作流**：
1. 选择"并发工作流"
2. 输入任务："审查这段用户认证代码"
3. 执行后查看三个审查员的独立意见

### 示例2：文档编写流程

**创建协作**：
```
名称：技术文档编写
描述：编写和审核技术文档
```

**添加Agent**：
- Agent1：技术文档编写员
- Agent2：技术文档审核员

**执行工作流**：
1. 选择"任务移交工作流"
2. 输入任务："编写API接口文档"
3. Agent1编写 → Agent2审核 → Agent1修改 → 最终完成

### 示例3：产品开发流程

**创建协作**：
```
名称：产品开发流水线
描述：从需求到测试的完整流程
```

**添加Agent**：
- Agent1：产品经理
- Agent2：架构师
- Agent3：开发工程师
- Agent4：测试工程师

**执行工作流**：
1. 选择"顺序工作流"
2. 输入任务："开发用户管理模块"
3. 按顺序执行：需求分析 → 架构设计 → 代码开发 → 测试验证

## 🎯 最佳实践

### 1. Agent配置建议

- **每个Agent应该有明确的职责**
  - 在创建Agent时设置清晰的System Prompt
  - 例如："你是一个专业的代码安全审查员"

- **为Agent选择合适的模型**
  - 复杂任务选择强大的模型（如GPT-4）
  - 简单任务选择快速的模型（如GPT-3.5）

### 2. 工作流选择建议

| 任务类型 | 推荐工作流 | 原因 |
|---------|-----------|------|
| 流水线任务 | 顺序工作流 | 有明确的依赖关系 |
| 独立审查 | 并发工作流 | 提高效率，获得多角度意见 |
| 迭代优化 | 任务移交 | 需要反复修改完善 |
| 团队讨论 | 群聊工作流 | 需要多方意见交流 |

### 3. 任务描述建议

- **清晰明确**：描述要具体，避免模糊
- **提供上下文**：包含必要的背景信息
- **明确输出格式**：说明期望的输出格式

**好的示例**：
```
请审查以下用户认证代码的安全性和性能：
1. 检查SQL注入风险
2. 检查密码存储方式
3. 检查会话管理
4. 提供改进建议
```

**不好的示例**：
```
审查代码
```

## 🔧 常见问题

### Q1: 协作列表为空怎么办？

**解决方案**：
1. 先在"协作管理"页面创建协作
2. 确保至少添加了一个Agent
3. 刷新协作工作流页面

### Q2: 工作流执行失败？

**可能原因**：
1. Agent没有配置大模型
2. 大模型API密钥无效
3. 网络连接问题

**解决方案**：
1. 检查Agent的大模型配置
2. 在"大模型配置"页面测试连接
3. 查看错误日志

### Q3: 如何查看执行过程？

**方法**：
1. 执行完成后，在结果区域查看"执行过程"
2. 每条消息显示：
   - 发送者（Agent名称）
   - 内容（Agent的回复）
   - 时间戳

### Q4: 群聊工作流没有响应？

**说明**：
- 群聊工作流使用流式响应
- 消息会实时推送到控制台
- 检查浏览器控制台查看实时消息

## 📊 数据流程图

```
用户操作
   ↓
创建协作（Collaboration）
   ↓
添加Agent到协作（CollaborationAgent）
   ↓
访问协作工作流页面
   ↓
选择协作 → 选择工作流类型 → 输入任务
   ↓
调用后端API（/api/collaborationworkflow/{id}/{type}）
   ↓
CollaborationWorkflowService执行工作流
   ↓
获取协作中的Agent列表
   ↓
通过AgentFactoryService创建Agent实例
   ↓
根据工作流类型执行：
  - Sequential: 顺序调用Agent
  - Concurrent: 并发调用Agent
  - Handoffs: 动态移交任务
  - GroupChat: 群聊协作
   ↓
返回执行结果
   ↓
前端展示结果
```

## 🚀 快速开始

1. **确保后端服务运行**
   ```bash
   cd backend/src/MAFStudio.Api
   dotnet run
   ```

2. **确保前端服务运行**
   ```bash
   cd frontend
   npm start
   ```

3. **访问系统**
   - 打开浏览器：http://localhost:3000
   - 登录系统

4. **创建第一个协作**
   - 侧边栏 → 协作管理 → 新建协作
   - 添加Agent
   - 侧边栏 → 协作工作流 → 执行工作流

**提示**：如果遇到任何问题，请查看浏览器控制台和后端日志获取详细错误信息。
