# 工作流模板系统设计

## 概述

工作流模板系统是一个用于保存、管理和复用工作流定义的系统。它允许用户将优秀的工作流保存为模板，方便下次直接使用或作为参考。该系统支持模板分类、标签管理、搜索过滤、使用统计等功能，并能够与 Magentic 工作流系统集成，实现自动学习和优化。

## 核心功能

### 1. 模板管理
- 创建模板
- 编辑模板
- 删除模板
- 复制模板

### 2. 模板分类
- 按分类组织（开发、设计、文档、测试等）
- 标签管理
- 公开/私有设置

### 3. 模板搜索
- 关键词搜索
- 标签过滤
- 分类过滤
- 公开/私有过滤

### 4. 模板执行
- 选择模板执行
- 参数配置
- 执行监控
- 结果保存

### 5. 使用统计
- 使用次数统计
- 热门模板排行
- 使用历史记录

## 数据库设计

### workflow_templates 表

```sql
CREATE TABLE workflow_templates (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(100),
    tags JSONB DEFAULT '[]'::jsonb,
    
    -- 工作流定义（JSON格式）
    workflow_definition JSONB NOT NULL,
    
    -- 参数定义
    parameters JSONB DEFAULT '{}'::jsonb,
    
    -- 元数据
    created_by BIGINT,
    is_public BOOLEAN DEFAULT false,
    usage_count INTEGER DEFAULT 0,
    
    -- Magentic学习相关
    source VARCHAR(50) DEFAULT 'manual',
    original_task TEXT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_workflow_templates_category ON workflow_templates(category);
CREATE INDEX idx_workflow_templates_created_by ON workflow_templates(created_by);
CREATE INDEX idx_workflow_templates_is_public ON workflow_templates(is_public);
CREATE INDEX idx_workflow_templates_tags ON workflow_templates USING GIN(tags);
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键ID |
| name | VARCHAR(255) | 模板名称 |
| description | TEXT | 模板描述 |
| category | VARCHAR(100) | 模板分类 |
| tags | JSONB | 标签（JSON数组） |
| workflow_definition | JSONB | 工作流定义（JSON格式） |
| parameters | JSONB | 参数定义（JSON对象） |
| created_by | BIGINT | 创建者ID |
| is_public | BOOLEAN | 是否公开 |
| usage_count | INTEGER | 使用次数 |
| source | VARCHAR(50) | 来源：manual、magentic、magentic_optimized |
| original_task | TEXT | 原始任务描述（如果是Magentic生成的） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 前端实现

### 1. 模板管理页面

```tsx
const WorkflowTemplateManagement: React.FC = () => {
  const [templates, setTemplates] = useState<WorkflowTemplate[]>([]);
  const [loading, setLoading] = useState(false);
  
  // 加载模板列表
  const loadTemplates = async () => {
    setLoading(true);
    try {
      const data = await workflowTemplateApi.getAll(publicFilter, categoryFilter);
      setTemplates(data);
    } finally {
      setLoading(false);
    }
  };
  
  // 删除模板
  const handleDelete = async (id: number) => {
    await workflowTemplateApi.delete(id);
    message.success('删除成功');
    loadTemplates();
  };
  
  return (
    <div>
      <Card title="工作流模板">
        <Table
          dataSource={templates}
          columns={[
            { title: '名称', dataIndex: 'name' },
            { title: '描述', dataIndex: 'description' },
            { title: '分类', dataIndex: 'category' },
            { title: '使用次数', dataIndex: 'usageCount' },
            {
              title: '操作',
              render: (record) => (
                <Space>
                  <Button onClick={() => handleEdit(record)}>编辑</Button>
                  <Button onClick={() => handleExecute(record)}>执行</Button>
                  <Button danger onClick={() => handleDelete(record.id)}>删除</Button>
                </Space>
              )
            }
          ]}
        />
      </Card>
    </div>
  );
};
```

### 2. 工作流编辑器

```tsx
const WorkflowEditor: React.FC = () => {
  const [nodes, setNodes, onNodesChange] = useNodesState([]);
  const [edges, setEdges, onEdgesChange] = useEdgesState([]);
  
  // 拖放处理
  const onDrop = useCallback((event: React.DragEvent) => {
    const type = event.dataTransfer.getData('application/reactflow');
    const position = { x: event.clientX, y: event.clientY };
    
    const newNode = {
      id: `node-${Date.now()}`,
      type,
      position,
      data: { name: getDefaultNodeName(type) }
    };
    
    setNodes((nds) => nds.concat(newNode));
  }, [setNodes]);
  
  // 保存工作流
  const handleSave = async (values: any) => {
    const workflow = {
      nodes: nodes.map(node => ({
        id: node.id,
        type: node.type,
        ...node.data
      })),
      edges: edges.map(edge => ({
        type: edge.data?.type || 'sequential',
        from: edge.source,
        to: edge.target
      }))
    };
    
    await workflowTemplateApi.create({
      ...values,
      workflow
    });
    
    message.success('保存成功');
  };
  
  return (
    <div style={{ display: 'flex', height: '100vh' }}>
      {/* 节点面板 */}
      <NodePanel />
      
      {/* 工作流画布 */}
      <ReactFlow
        nodes={nodes}
        edges={edges}
        onNodesChange={onNodesChange}
        onEdgesChange={onEdgesChange}
        onDrop={onDrop}
      >
        <Background />
        <Controls />
        <MiniMap />
      </ReactFlow>
      
      {/* 属性面板 */}
      <PropertyPanel selectedNode={selectedNode} />
    </div>
  );
};
```

## Magentic 集成

### 1. 自动学习机制

当 Magentic 工作流执行成功后，系统会：
1. 分析工作流执行效果
2. 提取任务特征和标签
3. 保存工作流定义
4. 记录原始任务描述
5. 标记来源为 `magentic_optimized`

### 2. 模板匹配

当用户输入新任务时，系统会：
1. 提取任务关键词和标签
2. 在模板库中搜索相似模板
3. 计算相似度分数
4. 推荐最匹配的模板

```csharp
public async Task<WorkflowTemplateDto?> FindMatchingTemplateAsync(string task)
{
    // 提取任务标签
    var taskTags = ExtractTags(task);
    
    // 搜索相似模板
    var templates = await _templateRepository.FindByTagsAsync(taskTags, minSimilarity: 0.8);
    
    // 返回最匹配的模板
    return templates.FirstOrDefault();
}
```

### 3. 模板优化

当用户修改 Magentic 生成的计划后：
1. 保存修改后的计划
2. 更新模板定义
3. 提高模板的相似度权重
4. 记录优化历史
